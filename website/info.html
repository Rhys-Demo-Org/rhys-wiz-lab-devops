NGINX + Flask Demo Website

Welcome to Your NGINX + Flask Server on AWS App Runner

This demo website showcases NGINX serving static content while proxying API requests to a Flask backend, all containerized on AWS App Runner. This architecture demonstrates both frontend and backend vulnerability scanning.

Learn About the Architecture

What is This Demo?

This application combines NGINX as a web server and reverse proxy with a Flask Python backend, demonstrating a common microservices architecture pattern.

NGINX Frontend

NGINX serves static HTML, CSS, and JavaScript files with high performance and low resource consumption.

Flask Backend

Python Flask API handles dynamic requests, demonstrating backend vulnerabilities in dependencies like Flask, Werkzeug, and Jinja2.

Reverse Proxy

NGINX proxies API requests to Flask on the backend, creating a unified application interface.

⚠️ Vulnerable Python Packages

Warning: The Flask backend uses intentionally outdated packages with known CVEs for security scanning demonstrations.

Vulnerable Packages:
• Flask 1.1.1 (CVE-2023-30861)
• Werkzeug 0.16.0 (CVE-2023-25577 - High Severity)
• Jinja2 2.10.1 (CVE-2020-28493 - ReDoS)
• requests 2.20.0 (CVE-2018-18074)
• urllib3 1.24.1 (Multiple CVEs)

Fix: Use Dockerfile.secure with updated packages

AWS App Runner Deployment

This website is deployed using AWS App Runner with CI/CD via GitHub Actions workflows. The container is stored in Amazon ECR and the DNS is managed through Route53.
